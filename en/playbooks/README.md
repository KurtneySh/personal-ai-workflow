# Playbooks

Playbooks are end-to-end workflows for setting up, changing, and handing off AI-readable projects. Use them when a task is bigger than a single checklist and needs staged judgment, agent collaboration, verification, and human gates.

Start here:

- [Start A New Project](start-a-new-project.md): turn a new idea or an existing repo into a workspace an agent can understand, verify, and safely continue.

Use the playbooks together with:

- [Guides](../guides/README.md) for project-level judgment and the smallest useful infra by level.
- [Loops](../loops/README.md) when you are deciding whether a task can repeat automatically, what the verification signal is, and when to stop / escalate.
- [Skill Workflow](../skill-workflow/README.md) when a repeated workflow may deserve a project-level skill or a system-level skill candidate.
